# 页面解析和数据提取
- 结构数据: 先有结构,再有数据
    - JSON文件
        - JSON Path
        - 转换成Python类型,在进行操作(json类)
    - xml 文件
        - 转换成python类型(xmltodic)
        - XPath
        - CSS选择器
        - 正则
- 非结构化数据: 先有数据,再有结构
    - 文本
    - 电话号码
    - 邮箱地址
        - 通常处理此类数据,使用正则表达式
    - html 文件   
        - 正则
        - XPath
        - css选择器
# 正则表达式
- 一套规则,可以再字符串文本中进行搜查替换等
- 案例v23,match的基本使用
- 正则常用的方法:
    - match: 从开始位置找,进行一次匹配
    - search: 从开始位置开始查找, 一次匹配, 案例24
    - findall: 全部匹配,返回列表
    - finditer: 全部匹配,返回迭代器
    - split: 分割字符串,返回列表
    - sub: 替换
